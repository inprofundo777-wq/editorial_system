# Constitution

Папка `Constitution` содержит документы, определяющие природу In Profundo, архитектуру Editorial System и первый цикл формирования нормативного корпуса.

Настоящий `README.md` является навигационным и статусным документом папки. Он не заменяет содержательные документы и не обладает самостоятельной нормативной силой.

---

# 1. Основной принцип работы с документами

Работа над постоянными документами проходит по следующему циклу:

1. документ обсуждается и разрабатывается в профильной рабочей ветке;
2. после согласования создаётся полный итоговый файл в формате Markdown;
3. файл переносится в GitHub;
4. ссылка возвращается для проверки;
5. после проверки GitHub-файл становится действующим источником истины;
6. дальнейшая работа ведётся по GitHub-версии, а не по истории чата;
7. при выпуске новой редакции прежняя версия сохраняется в архиве с точным номером версии;
8. постоянное имя действующего документа не меняется, чтобы не нарушать внутренние ссылки.

Пустой файл, заготовка или документ без указанного статуса не является нормативным документом.

# 5. Правила статусов

В этой папке используются следующие основные статусы:

- **действующий источник истины** — документ регулирует текущую работу;
- **действующая рабочая редакция** — документ используется, но предусматривает дальнейший review;
- **утверждённый архитектурный документ** — принятое основание для текущего цикла формирования системы;
- **проект** — текст подготовлен для рассмотрения, но не утверждён;
- **мастер-проект** — исходный расширенный текст, сохраняющий происхождение будущих документов;
- **запланирован** — документ предусмотрен архитектурой, но ещё не создан;
- **пустая заготовка** — файл существует технически, но не является документом;
- **на пересмотре** — действующий или проектный текст проходит содержательную проверку;
- **заменён** — документ утратил действующую силу после выхода новой редакции;
- **архивный** — сохраняется как историческое свидетельство и источник происхождения, но не регулирует текущую работу.

# 6. Именование и версии

Постоянный путь сохраняется за действующей редакцией документа.

Пример:

```text
Constitution.md
```

При выпуске новой редакции предыдущая версия сохраняется в архиве:

```text
archive/Constitution-v1.0.md
```

После этого новый действующий текст остаётся по постоянному пути:

```text
Constitution.md
```

Аналогичный принцип применяется к другим постоянным документам:

```text
DNA.md
ARC-001.md
FRM-001-Architecture-and-Authority.md
POL-001-Documents-Decisions-Archive.md
```

Изменение номера версии фиксируется в паспорте документа и истории изменений.

# 7. Архитектурное примечание

Не каждый существующий файл должен стать самостоятельным нормативным документом.

Документ создаётся или сохраняется отдельно только тогда, когда имеет:

- самостоятельную функцию;
- определённый нормативный уровень;
- ясно ограниченную сферу действия;
- связь с документами более высокого уровня;
- необходимость, которую нельзя лучше удовлетворить уже существующим документом.

Поэтому пустые файлы `Core_Principles.md`, `Editorial_Philosophy.md`, `Language.md` и `Mission.md` не должны наполняться только потому, что они уже существуют. Их судьба определяется после завершения Конституции 1.0 и первого нормативного корпуса.
