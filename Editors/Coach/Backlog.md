
# Backlog

## Purpose

Настоящий документ содержит наблюдения, идеи, гипотезы и предложения по развитию роли Editorial Coach, которые ещё не стали частью постоянной архитектуры.

Backlog не является планом работ и не изменяет поведение роли.

Каждая запись рассматривается как материал для дальнейшего наблюдения и анализа.

---

# Rules

Backlog используется для хранения:

- повторяющихся наблюдений;
- возможных архитектурных улучшений;
- открытых вопросов;
- гипотез;
- идей новых документов;
- предложений по изменению существующих документов;
- предложений по созданию новых редакционных ролей.

Backlog не содержит утверждённых решений.

# Review

После завершения одного или нескольких рабочих циклов каждая запись получает одно из решений:

- Promote to Development
- Keep in Backlog
- Reject

До принятия решения ни одна запись не изменяет архитектуру Editorial System.
