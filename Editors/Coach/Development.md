# Development

## Purpose

Настоящий документ содержит изменения роли Editorial Coach, утверждённые к реализации после рассмотрения Backlog.

Development является рабочим планом развития роли.

Все изменения настоящего документа считаются принятыми, но ещё не интегрированными в постоянную документацию.

---

# Rules

Каждая запись должна:

- ссылаться на Backlog;
- содержать обоснование изменения;
- определять документы, которые необходимо обновить;
- считаться завершённой только после внесения изменений во все соответствующие документы.

**Reason**

После создания Backlog.md и Development.md документация должна отражать полный цикл:

- определить роль Backlog Review;
- определить момент завершения изменения;
- проверить согласованность с Constitution.md и Development.md.

Approved

# Integration

После завершения реализации запись:

- удаляется из Development;
- отражается в соответствующих документах роли;
- при необходимости фиксируется в истории изменений.

Development не используется для хранения идей или гипотез.

Для этого предназначен Backlog.
